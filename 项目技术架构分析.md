## 技术之间的联系

### 1. 请求处理流程

```
用户请求 → Filter/Interceptor → Controller → Service → Mapper → Database
                                      ↓
                              Sa-Token认证 ← Redis缓存
```

### 2. AI对话流程（qc-aigc）

```
用户消息 → ChatController → CozeChatService/SpringAI
                                      ↓
                    Vector Store (Redis)  ←  知识库检索
                                      ↓
                          AI模型（Ollama/OpenAI/Coze）
                                      ↓
                         SSE流式响应 → 前端实时显示
```

### 3. Aiflowy AI工作流

```
User Request → TinyFlow (工作流引擎)
                    ↓
         AgentsFlex (Agent执行框架)
                    ↓
         MCP协议 ←→ 外部工具 (课程查询等)
                    ↓
         AI模型 (通义千问/其他)
                    ↓
         WebSocket/SSE 响应
```

### 4. 认证与会话

```
登录请求 → Sa-Token → 生成Token → 存入Redis
                                      ↓
后续请求 → Token验证 → Redis查询 → 权限校验 → 业务处理
```

## 技术选型亮点

1. **多ORM支持**：根据不同项目选择MyBatis-Plus或MyBatis-Flex，都支持lambda查询
2. **AI多模式支持**：同时支持Ollama（本地）、OpenAI（云端）、Coze（第三方）多种AI接入
3. **工作流引擎**：TinyFlow提供AI任务编排能力
4. **文件存储抽象**：X-File-Storage统一管理本地/OSS等多种存储方式
5. **微服务友好**：Sa-Token支持分布式会话，Redis统一缓存
